Vi-Vim-键盘图及其基本命令

发表于 2017-04-16 | | 阅读次数

VIM键盘图

Vim的几种工作模式定义

便于理解的区分模式

正常模式

可以使用快捷键命令，或按:输入命令行

插入模式

i、a、o、O -> 插入模式

可视模式

v -> 可视模式
V -> 可视行模式
ctrl+v -> 可视块模式

替换模式

R -> 替换模式

绕口的vi的工作模式

命令行模式 （command mode）

控制屏幕光标的移动，字符、字或行的删除，移动复制某区段及进入Insert mode下，或者到 last line mode。

插入模式（Insert mode）

只有在 Insert mode 下，才可以做文字输入，按「ESC」键可回到命令行模式。

底行模式（last line mode）

将文件保存或退出 vi，也可以设置编辑环境，如寻找字符串、列出行号等。

常用命令

使用Vim打开文件

  • vi filename：打开或新建文件，并将光标置于第一行首

  • vi +n filename：打开文件，并将光标置于第n行首

  • vi + filename：打开文件，并将光标置于最后一行首

  • vi -r filename：如果在上次使用vi编辑时发生系统崩溃，恢复文件

  • vi -R filename：以只读的方式打开文件，但可以强制保存

  • vi -M filename：以只读的方式打开文件，且不可以强制保存

  • vi file1....filen：打开多个文件，依次编辑，如vi file0.txt file1.txt，输入:n切换到下一个文件，输入:N切换到上一个文件

  • vi +/String filename：打开文件，并将光标置于找到的第一个String行上，String区分大小写

文档操作

  • :e file -> 关闭当前编辑的文件，并开启新的文件。 如果对当前文件的修改未保存，vi会警告
  • :e! file -> 放弃对当前文件的修改，编辑新的文件
  • :e + file -> 开始新的文件，并从文件尾开始编辑
  • :e +n file -> 开始新的文件，并从第n行开始编辑
  • :enew -> 编译一个未命名的新文档
  • :e -> 重新加载当前文档
  • :e! -> 重新加载当前文档，并丢弃已做的改动
  • :e# -> 回到刚才编辑的文件(如在使用:e a.txt命令后，输入:e#返回刚才编辑的文件)
  • :f(或Ctrl + g) -> 显示文档名，是否修改，和光标位置
  • :f filename -> 改变编辑的文件名，这时再保存相当于另存为
  • :n1,n2w filename -> 选择性保存从某n1行到另n2行的内容
  • :saveas newfile -> 另存为newfile

命令行模式

控制光标的移动的命令

  • k、j、h、l -> 上、下、左、右
  • +或Enter -> 光标移动至下一行第一个非空字符
  • - -> 光标移动至上一行第一个非空字符
  • Ctrl+b -> 向前翻一页(PageUp)
  • Ctrl+f -> 向后翻一页(PageDown)
  • ctrl+d -> 向前翻半页
  • ctrl+u -> 向后翻半页
  • Ctrl+e -> 向下滚动一行
  • Ctrl+y -> 向上滚动一行
  • H -> 光标置于当前屏幕最上行(Highest)
  • nH -> 光标置于当前屏幕的第n行(如2H将光标移到屏幕的第2行)
  • M -> 光标置于当前屏幕中间(Middle)
  • L -> 光标置于屏幕最后一行(Lowest)
  • nL -> 光标置于屏幕的倒数第n行(如3L将光标移到屏幕的倒数第3行)
  • w -> 小写字母w，在指定行内右移光标，到下一个字的开头
  • W -> 大写字母W，在指定行内右移光标，到下一个字的开头，忽略部分标点，如逗号
  • e -> 在指定行内右移光标，到一个字的末尾
  • b -> 在指定行内左移光标，到前一个字的开头
  • B -> 在指定行内左移光标，到前一个字的开头，忽略部分标点，如逗号
  • 0 -> 数字0，左移光标到本行的开头
  • ^ -> 移动光标，到本行的第一个非空字符
  • G -> 光标移动到文章的最后
  • gg -> 光标移动到文章的开始
  • nG -> 光标移动到文章的第n行(如8G移动到文章的第8行)
  • $ -> 右移光标，到本行的末尾
  • fc -> 光标移动至同一行的下一个c字符处
  • Fc -> 光标移动至同一行的上一个c字符处
  • tc -> 光标移动至同一行的下一个c字符前
  • Tc -> 光标移动至同一行的上一个c字符后
  • n| -> 光标移动第n列

替换和删除

  • rc -> 用c替换光标所指向的当前字符(r:replace)
  • nrc -> 用c替换包含光标指向字符向后的n个字符(如5rc用c替换包含光标指向字符的后5个字符，举例:有abcdefg这些字符，光标指向d，输入3rn，得到abcnnng)
  • x -> 删除光标所在位置的字符
  • nx -> 删除包含光标所在位置后面的n个字符(如3x删除光标所在位置后面的3个字符)
  • X -> 大写的X，删除光标所在位置(不包含光标所在位置的字符)前面的一个字符
  • nX -> 大写的X，删除光标所在位置(不包含光标所在位置的字符)前面的n个字符(如3X删除光标所在位置前面的3个字符)
  • dd -> 删除光标所在行，并去除空隙(删除的行可被p命令粘贴出来)
  • ndd(或dnd) -> 从光标所在行开始向后删除n行内容(包括光标所在行)，并去除空隙(如3dd删除3行内容，并去除空隙）
  • d[n]w -> 向后删除(剪切)n个单词
  • d[n]l -> 向右删除(剪切)n个字符
  • d[n]h -> 向左删除(剪切)n个字符

复制和粘贴

可被粘贴的原因

从正文中删除的内容（如字符、字或行）并没有真正丢失，而是被剪切并复制到了一个内存缓冲区中。用户可将其粘贴到正文中的指定位置。

  • p -> 小写字母 p，将缓冲区的内容粘贴到光标的后面
  • P -> 大写字母 P，将缓冲区的内容粘贴到光标的前面

注意:

如果缓冲区的内容是字符或字，直接粘贴在光标的前面或后面；如果缓冲区的内容为整行正文则粘贴在当前光标所在行的上一行或下一行。

  • yy或Y -> 复制当前行到内存缓冲区
  • nyy -> 复制n行内容到内存缓冲区(如5yy复制5行内容到内存缓冲区)
  • y -> 复制在可视模式下选中的文本
  • y[n]w -> 复制n个词
  • y[n]l -> 复制光标右边n个字符
  • y[n]h -> 复制光标左边n个字符
  • y$ -> 从光标当前位置复制到行尾
  • y0 -> 从光标当前位置复制到行首
  • ygg或y1G -> 复制光标所在行以上的所有行(包括光标所在行)
  • yG -> 复制光标所在行以下的所有行(包括光标所在行)
  • yaw -> 复制一个词，光标可以不在词首

搜索字符串

  • /str1 -> 正向搜索字符串str1
  • ?str2 -> 反向搜索字符串str2
  • n -> 继续搜索，找出str1字符串下次出现的位置

撤销和重复

  • u -> 撤消前一条命令的结果
  • . -> 重复最后一条修改正文的命令

文本选中

  • v -> 字符选中命令
  • V -> 行选中命令
  • Ctrl + v -> 块选中命令

插入模式

进入插入模式

  • i -> 在光标左侧输入正文
  • a -> 在光标右侧输入正文
  • o -> 小写字母o，在光标所在行的下一行增添新行
  • O -> 大写字母O，在光标所在行的上一行增添新行
  • I -> 在光标所在行的开头输入正文
  • A -> 在光标所在行的末尾输入正文

退出插入模式

  • ESC键或组合键Ctrl+[

底行模式

退出命令

  • :q -> 在未作修改的情况下退出
  • :q! -> 放弃所有修改，退出编辑程序

行号和文件保存

  • :n ->将光标移到第n行
  • :set nu -> 显示行号
  • :set nonu -> 取消行号显示

底行模式下，可以规定命令操作的行号范围。数值用来指定绝对行号；字符“.”表示光标所在行的行号；字符“$”表示正文最后一行的行号；简单的表达式，例如“.+5”表示当前行往下的第 5 行。例如：

  • :.+5 -> 将光标移到当前行之后的第5行
  • :$ ->将光标移到正文最后一行

在底行模式下，允许从文件中读取正文，或将正文写入文件。例如：

  • :w -> 将编辑的内容写入原始文件，用来保存编辑的中间结果
  • :wq或:x或ZZ -> 将编辑的内容写入原始文件并退出编辑程序
  • :w file -> 将编辑的内容写入file文件，保持原有文件的内容不变
  • :a,bw file -> 将第 a 行至第 b 行的内容写入 file 文件 （如:1,.w file 将第 1 行至当前行写入 file 文件）
  • :r file -> 读取 file 文件的内容，插入当前光标所在行的后面
  • :nr file -> 读取 file 文件的内容，在第n行插入
  • :f file -> 将当前文件重命名为 file
  • :r !date -> 在光标处插入当前日期和时间
  • :c[n]w -> 改写光标后n个单词
  • :c[n]l或[n]s -> 改写光标后n个字母
  • :c[n]h -> 改写光标前n个字母
  • :[n]cc -> 修改当前[n]行

字符串搜索

  • :/str/ -> 正向搜索，将光标移到下一个包含字符串 str 的行
  • :?str? -> 反向搜索，将光标移到上一个包含字符串 str 的行
  • :/str/+number -> 光标停在包含str字符串的行后面的第number行上
  • :/str/-number -> 光标停在包含str字符串的行前面的第number行上

正文替换

  • :s/str1/str2/ -> 用字符串 str2 替换当前行中首次出现的字符串 str1
  • :s/str1/str2/g -> 用字符串 str2 替换当前行中所有出现的字符串 str1
  • :.,$ s/str1/str2/g -> 用字符串 str2 替换正文中当前行到文章末尾所有出现的字符串 str1
  • :1,$ s/str1/str2/g -> 用字符串 str2 替换正文中所有出现的字符串 str1
  • :g/str1/s//str2/g -> 功能同:1,$ s/str1/str2/g

从上述替换命令可以看到：g 放在命令末尾，表示对搜索字符串的每次出现进行替换；不加 g，表示只对搜索字符串的首次出现进行替换；g 放在命令开头，表示对正文中所有包含搜索字符串的行进行替换操作。

  • :%s/str1/str2/g -> 用str2替换正文中所有的str1
  • :%s/^/str2/g -> 正文的每行行首添加str2字符串

所有替换命令的末尾添加c命令，每个替换操作都需要用户确认，如:%s/str1/str2/gc，加上i则忽略大小写，如:$s/STR1/str2/gci

删除正文

  • :d -> 删除光标所在行
  • :3d -> 删除第 3 行
  • :.,$d -> 删除当前行至正文的末尾
  • :/str1/,/str2/d -> 删除从字符串 str1 到 str2 的所有行

恢复文件

vi 在编辑某个文件时，会另外生成一个临时文件，这个文件的名称通常以 . 开头，并以 .swp 结尾。vi 在正常退出时，该文件被删除，若意外退出，而没有保存文件的最新修改内容，则可以使用恢复命令，也可以在启动 vi 时利用 -r 选项。

  • :recover -> 恢复文件

选项设置

为控制不同的编辑功能，vi 提供了很多内部选项。利用 :set 命令可以设置选项。基本语法为：

  • :set option -> 设置选项 option

常见的功能选项包括：

autoindent -> 设置该选项，则正文自动缩进
ignorecase -> 设置该选项，则忽略规则表达式中大小写字母的区别
number -> 设置该选项，则显示正文行号
ruler -> 设置该选项，则在屏幕底部显示光标所在行、列的位置
tabstop -> 设置按 Tab 键跳过的空格数。例如 :set tabstop=n，n 默认值为 8
mk -> 将选项保存在当前目录的 .exrc 文件中

拼写检查

  • :set spell -> 开启拼写检查
  • :set nospell -> 关闭拼写检查
  • ]s -> 移动到下一个拼写错误的单词

shell切换

在编辑正文时，利用 vi 底行模式下提供的shell切换命令，无须退出vi即可执行Linux命令

  • :! command -> 执行完shell命令command后回到vi，如:! uname -a

另外，在命令模式下，键入K(直接输入大写字母K，不带:) ，可命令 vi 查找光标所在单词的手册页，相当于运行man命令。

vim中文本对象

  • aw -> 一个词
  • as -> 一句
  • ap -> 一段
  • ab -> 一块

vim排版命令

  • >> -> 向右缩进一个shiftwidth
  • << -> 向左缩进一个shiftwidth
  • :ce -> 本行文字居中
  • :le -> 本行文字居左
  • :ri -> 很行文字居右
  • J -> 拼接当前行和下一行
  • gJ -> 拼接当前行和下一行，合并后不留空格
